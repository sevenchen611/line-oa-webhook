每 15 分鐘檢查命令

這個檢查負責處理從 LINE 進來、需要 Codex 或後端接手的即時指令。

15 min Codex Command Queue

檢查項目

項目檢查內容具體做法
命令佇列 檢查 Codex Command Queue 中是否有新的 LINE 指令。 LINE webhook 收到「Seven Junior，我們來做任務校準」或其他非即時可處理指令時，會建立命令佇列項目，等待 Codex 或指定流程接手。
即時可處理命令 辨識可以立即回覆的查詢。 目前即時處理以「查待辦」與「打開第幾個任務」為主，會直接回 LINE；這類操作指令保留為命令紀錄，不建立總控任務。
需 Codex 接手命令 辨識需要開發、資料庫修改、任務校準或人工判斷的指令。 將指令、來源群組、發話者、LINE 訊息 ID、風險等級與原文寫進命令佇列，供 Codex 後續執行。
處理狀態 確認命令是否仍待處理、已處理、失敗或需要人工確認。 命令處理流程可讀取佇列並更新狀態，避免同一個指令被重複處理，也讓 Seven 可以知道哪些指令還在等 Codex。

命令與任務的邊界

查詢命令例如查待辦、列出今天待辦、打開第 2 個任務。這是系統操作，不是現實世界待辦。
校準命令例如「我們來做任務校準」。這是要求 Codex 開始一個操作流程，應進入命令佇列，不直接變成總控任務。
真實任務若同一則訊息除了命令之外，還包含對外承諾、負責人、交付物、截止日或風險事件，才需要另依任務規則判斷是否建立任務。
紀錄位置命令保存在 Codex Command Queue 或 LINE 訊息紀錄；真實任務才進入總控任務庫。
這個 15 分鐘檢查的價值，是讓「操作 Seven Jr.」和「建立真實待辦」分開處理。這樣任務庫不會被查詢指令污染，但 Codex 仍能接到需要處理的工作。
