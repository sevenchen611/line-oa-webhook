每日定時報告與更新說明

SevenAM 每天固定推送早報、追蹤更新與每日總控報告，讓任務、決策與進度能被定時校準。

Daily reports LINE + Notion

排程總覽

時間報告名稱程式名稱具體執行內容
08:00 早上 8 點報告 npm run cron:report -- morning 產生早晨總控報告，提醒今日行程、優先工作、未完成事項、需要決策的項目，以及新增或尚未確認任務是否需要先做目標追認。
10:00 上午追蹤更新 npm run cron:report -- followup-morning 檢查上午新增任務、專案與目標定義缺口。若任務沒有完成目標、驗收方式或負責人口述，提醒先補目標再列入真實進度。
13:00 午間追蹤更新 npm run cron:report -- followup-midday 整理午間前新增的待確認任務、追蹤訊息、需要退回修改的項目，以及上午未處理完的確認事項。
17:00 下午追蹤更新 npm run cron:report -- followup-afternoon 確認下午前的任務變動、卡點、等待回覆、需要 Seven 決策或需要外部提醒的事項，準備晚間總控報告的材料。
20:30 晚上 8 點半每日總控報告 npm run cron:report -- daily 彙整當日主要事件、LINE 原始訊息、判斷產生的任務、專案進度報告與需要追蹤的風險，形成每日收斂報告。

早上 8 點報告內容

今日焦點列出今天最需要 Seven 注意的任務、會議、決策與未完成事項。
目標追認對新增或尚未確認的任務，要求先問清楚完成目標、怎樣叫完成、由誰驗收。
任務整理提示待確認、待回覆、待完成、可能需要退回的任務。
後續動作將使用者確認結果寫入 Notion，作為後續每小時檢查與日報判斷的依據。

10 點、13 點、17 點更新內容

時段處理重點輸出結果
10:00上午新增任務、尚未確認目標、需要先問負責人的項目。產出追蹤確認頁，讓 Seven 決定哪些提醒可送出、哪些任務要退回或修改。
13:00午間前新訊息與上午未閉環事項。整理待確認任務、追蹤訊息、需要修改或停止的項目。
17:00下午累積的任務變更、卡點、等待回覆與晚報素材。把日間任務進度收斂，提供 20:30 每日總控報告使用。

20:30 每日總控報告格式

摘要列出今日整合幾件主要事件，並說明報告範圍。
主要事件每件事以「主題、對象、影響、結論、建議待辦」呈現。
解法對高風險或高影響事件，提出下一步做法或管理機制。
深看標示此事件由幾則 LINE 訊息、任務或進度紀錄合併而來。
待確認列出需要 Seven 判斷是否成立、是否完成、是否要退回或是否要外部回覆的事項。
