## Purpose

This improvement fixes the gap between "recording many messages" and "understanding what should be done next."

Before this version, the Assistant Manager flow could capture many LINE messages and create many small tasks, but the 20:30 report was too fragmented. It listed message-level details such as separate comments about insurance, meeting attendance, family health, or tenant issues. The user had to mentally merge them into events.

This version changes the 20:30 report into event-level control:

1. Identify whether multiple messages belong to the same event.
2. Summarize the event into a clear conclusion.
3. State the subject, target, impact, conclusion, and recommended follow-up task.
4. Convert the recommended follow-up into the total control task database so future reports can track completion.

## Key Problems Found

| Problem | Why it mattered | Fix |
| --- | --- | --- |
| Message-level reporting was too noisy | The report was technically complete but hard to read. | Report now groups messages into events. |
| Important events were split into many tasks | Fire insurance, tenant issues, and meeting attendance became separate fragments. | Event synthesis merges same-event signals. |
| Reports did not show conclusion/action clearly | User needed a direct answer about what to do next. | Each event now shows subject, target, impact, conclusion, and recommended task. |
| Some events lacked follow-up tracking | Report text alone does not ensure completion. | Event-conclusion tasks were added to `總控任務庫`. |
| Relationship/commitment issues were under-detected | Wu case / slow finance response was not clearly surfaced. | Relationship escalation and finance-response patterns were added. |
