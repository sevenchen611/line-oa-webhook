# Upgrade Version Record

Version: `AM-IMP-2026.0608.02`

Improvement name: Scheduled report multi-recipient rule

Target project: 7AM

Upgrade date: 2026-06-08

Owner: Seven / Codex

## Purpose

Allow scheduled SevenAM reports to be sent to the main owner plus optional CC recipients, while still using SevenAM's own LINE conversation records and environment variables.

## Scope

| Item | Included? | Notes |
| --- | --- | --- |
| Code changes | Yes | Report target resolution now supports multiple env IDs and Notion keyword lookup. |
| Documentation changes | Yes | README, manifest, and this upgrade record updated. |
| Environment variables | Optional | Adds `SEVEN_REPORT_TARGET_IDS`, `SEVEN_REPORT_CC_TARGET_IDS`, and `SEVEN_REPORT_CC_NAME_KEYWORDS`. |
| Notion schema changes | No | Uses existing Seven LINE conversation master database. |
| Render sync/deploy | Yes | GitHub push completed and production control health / preview verified on Render. |

## Files Changed

| File | Change |
| --- | --- |
| `src/control-api.js` | `resolveReportTargets` now merges direct body targets, env primary recipients, env CC recipients, Notion primary keyword matches, and Notion CC keyword matches, then de-duplicates. |
| `render.yaml` | Added optional report recipient env var placeholders. |
| `README.md` | Documented the multi-recipient settings. |
