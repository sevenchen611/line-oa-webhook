## Purpose

Define the hourly 08:00-22:00 LINE check as task reconciliation rather than
message-level task creation.

Each hourly check should answer one core question:

```text
Is this new LINE message an update/evidence for an existing task, or a new event
that needs a new task?
```

## Behavior Contract

For every hourly LINE judgement run:

1. Read new LINE messages.
2. Review earlier context in the same LINE conversation.
3. Search active `總控任務庫` tasks for related event/task records.
4. Update the existing task when the message extends, answers, completes,
   blocks, changes, or clarifies it.
5. Create a new task only when the message is a genuinely new event.
6. Mark background, acknowledgement, duplicate, test, pure knowledge-sharing, or
   non-actionable record messages as judged without task creation.
