# UPGRADE-2026-06-14-AM-IMP-2026.0614.01

Project: SevenAM (SevenAM-specific feature; HOZO_AM not in scope)

Version: `AM-IMP-2026.0614.01`

## Summary

Rebuilt the 08:30 morning brief (`/reports/morning-brief`) as a live, data-driven
report. Previously this route always served a frozen static prototype
(`reports/morning-brief-prototype.html`, hardcoded to 2026-06-07 with sample
vendors), while the 10:00 follow-up was already dynamic — which is why the morning
brief looked old every day. The schedule (今日行程) is now read from the **real
Google Calendar across all of the controller's calendars** (Taiwan holidays kept,
foreign-holiday calendars excluded); carryover / priorities / decisions come from
Notion 總控任務. Cover date is always "today" (Asia/Taipei).

## Architecture (why a snapshot, not live render)

The page is served by Render, which has Notion/LINE creds but **no Google
credentials** (by design — Google secrets stay off the cloud). The resident local
worker has the Google refresh token. So the worker renders the morning brief HTML
locally at 08:30 and uploads it to Render, which stores it in Postgres and serves
the latest snapshot. No Google secret is added to Render.

Flow: worker `generate-morning-brief.js` → `POST /control/reports/snapshot`
(control-key auth) → Postgres `report_html_snapshots` → `GET /reports/morning-brief`
serves the latest `morning` snapshot (falls back to the prototype if none/DB error).

## Changes

- `scripts/morning-brief.js` (new): `fetchTodayCalendarEvents` (all calendars,
  dedupe, sort; excludes foreign-holiday calendars, keeps Taiwan), `fetchTasks`
  (Notion 總控任務), `buildMorningBrief()` → full mobile-friendly HTML with today's
  cover date. No hardcoded sample data.
- `scripts/generate-morning-brief.js` (new): worker entry. Renders + uploads the
  snapshot to Render. `--dry-run` writes `reports/morning-brief-preview.html`
  locally without uploading; `--print` prints the summary JSON.
- `src/report-snapshot-store.js` (new, Render side): Postgres-backed
  `saveReportSnapshot` / `getLatestReportSnapshot` (table `report_html_snapshots`,
  30-day retention). Lazy pool from `DATABASE_URL`; no-DB → disabled (fallback).
- `src/control-api.js`: `GET /reports/morning-brief` now serves the worker snapshot
  (prototype fallback); new `POST /control/reports/snapshot` (control-key auth).
- `scripts/local-worker.js`: at the 08:30 `morning` slot, runs
  `generate-morning-brief.js` before sending the LINE link, so the snapshot is ready
  when the controller opens it.
