# Upgrade Version Record

Version: `AM-IMP-2026.0608.08`

Improvement name: Hierarchical responsibility owner narrowing workflow

Target project: 7AM

Upgrade date: 2026-06-08

Owner: Seven / Codex

## Purpose

This improvement fixes the responsibility-assignment problem that appears when a
control center has many LINE conversations and many possible people.

Before this version, the responsibility table could ask the user to choose a
LINE group or person from a broad global list. That is acceptable with a small
number of records, but it does not scale. Once the system has hundreds or
thousands of conversations and users, choosing the correct owner from the full
set becomes too slow and error-prone.

This version changes the responsibility selection model into a layered flow:

1. Choose the total-control project.
2. Show only LINE groups assigned to that project.
3. Choose the main LINE group.
4. Show only known members of that group.
5. Choose the main owner and optional backup owner.

This lets Seven assign responsibility through visible names while the system
keeps the underlying Group ID and User ID for later follow-up, report, and LINE
push behavior.

## Rollback Plan

If this improvement causes wrong responsibility mappings:

1. Disable or remove the `seven-jr-responsibility-candidate-sync` cron job from
   `render.yaml`, then redeploy or resync the Blueprint.
2. Stop running `npm run responsibility:sync` manually.
3. Keep the existing responsibility rows intact; do not delete option rows or raw
   LINE conversation records.
4. If needed, revert the responsibility table view to manual group/person
   selection while preserving the `LINE對象名稱（結果）`,
   `LINE對象類型（結果）`, and `LINE對象ID（結果）` fields.
5. Revert `scripts/sync-responsibility-candidates.js`, `package.json`,
   `render.yaml`, and the related documentation from GitHub.
